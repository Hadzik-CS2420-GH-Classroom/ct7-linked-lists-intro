👨‍💻 Code-Together 7 — Singly Linked List Introduction

🎯 Overview

In this in-class activity, you’ll build a singly linked list from scratch using a Farr’s Ice Cream ticket queue scenario. You’ll complete TODO items in two source files while we walk through concepts together using discussion comments and diagrams.

📖 Learning Objectives

By completing this activity, you will be able to:

  • Create a Node struct with data and a next pointer for building linked chains
  • Implement push_front and push_back to insert nodes at the front and back of a list
  • Implement pop_front and pop_back to remove nodes from the front and back
  • Use the temp pointer pattern (save, advance, delete) to safely free nodes
  • Use the trailing pointer pattern (previous/current) to find the second-to-last node
  • Explain O(1) vs O(n) tradeoffs between front and back operations on a singly linked list
  • Write a proper destructor that traverses and frees every node to prevent memory leaks

📝 What You’ll Do

Open SinglyLinkedList.cpp and main.cpp, then complete the TODO comments. Each TODO tells you what to write, and the DISCUSSION comments explain the “why” behind the code. We’ll work through these together in class.

📚 Teaching Order

Work through SinglyLinkedList.cpp in the order main.cpp calls the operations:

1. Destructor — Why Manual Cleanup Matters (3 TODOs)

Concept What You’ll Learn
Temp pointer pattern Save, advance, delete — why order matters when freeing nodes
Traversal loop Walking the chain with while (head_) to delete every node

2. push_back — Customers Join the Line (5 TODOs)

Concept What You’ll Learn
Empty list special case New node becomes head when the list is empty
Traversal to last node Following next pointers to find the tail — O(n) cost

3. push_front — VIP Cuts to the Front (2 TODOs)

Concept What You’ll Learn
O(1) insertion One-liner: create node, point to old head, update head — no traversal needed
Array comparison Arrays must shift every element for front insertion — O(n)

4. pop_front — Serving the Next Customer (4 TODOs)

Concept What You’ll Learn
Temp pointer pattern (again) Same save-advance-delete pattern as the destructor
STL convention Why pop returns void — read first, then remove

5. pop_back — Customer at the Back Gives Up (5 TODOs)

Concept What You’ll Learn
Single-node special case When head is the only node, delete it and set head to nullptr
Trailing pointer pattern previous and current advance together to find the second-to-last node
O(n) cost Must traverse entire list — key weakness vs pop_front

6. print — Traversing the List (2 TODOs)

Concept What You’ll Learn
Fundamental traversal current starts at head, follows next until nullptr

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files in the include/ folder

Running the Program

  1. In Visual Studio, select ct7-linked-lists-intro.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. You should see output from each section as you complete the TODOs
Tip: Look for the // ! DISCUSSION: comments in each file — these explain the “why” behind each concept. The // ? SEE DIAGRAM: comments point you to visual aids in the images/ folder.
